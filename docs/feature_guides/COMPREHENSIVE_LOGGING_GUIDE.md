Alerts\n\n### Key Metrics to Monitor\n\n1. **Error Rate**: `(ERROR + CRITICAL logs) / Total logs * 100`\n2. **Response Times**: Track API call durations\n3. **User Activity**: Track user actions and Backend\n1. **Use structured logging** for all mutations\n2. **Include request_id** for request tracing\n3. **Sanitize sensitive data** (passwords, tokens)\n4. **Use appropriate log levels**\n5. **Include context** (user_id, entity info)\n\n### Frontend\n1. **Log user interactions** that affect state\n2. **Track navigation patterns** for UX insights\n3. **Monitor API performance** and errors\n4. **Avoid logging sensitive information**\n5. **Use batching** to avoid performance impact\n\n### Performance\n1. **Async logging** - don't block operations\n2. **Log rotation** to prevent disk space issues\n3. **Database cleanup** - remove old logs regularly\n4. **Selective logging** - avoid debug logs in production\n\n## рџљЁ Troubleshooting\n\n### Common Issues\n\n1. **Logs not
