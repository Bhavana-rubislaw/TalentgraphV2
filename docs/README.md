## 📝 Documentation Maintenance

### Adding New Documentation
1. Place files in the appropriate category folder
2. Update this README.md with the new file
3. Follow naming convention: `FEATURE_TYPE_DESCRIPTION.md`
4. Add to the relevant section above

### Documentation Standards
- **Markdown format** for all documentation
- **Clear headings** with hierarchy (H1 → H6)
- **Code examples** with syntax highlighting
- **Screenshots** where applicable (stored in `/docs/images/`)
- **Table of contents** for files >500 lines
- **Version history** at bottom of major docs
